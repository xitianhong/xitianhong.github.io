z-index翻译

发表于 2018-04-23 | 更新于 2018-05-03
Name z-index
Value auto丨丨inherit
Initial auto
Applies to positioned elements
Inherited no
Percentages N/A
Media visual
Computed value as specified

For a positioned box, the ‘z-index’ property xspecifies:

对于一个定位盒子，z-index属性指定了：

  1. The stack level of the box in the current stacking context.

    当前层叠上下文的层叠等级

  2. Whether the box establishes a stacking context.

    盒子是否创建一个层叠上下文

    Values have the following meanings:

    值有如下含义

This integer is the stack level of the generated box in the current stacking context. The box also establishes a new stacking context.

整数指定了 生成盒在当前层叠上下文中的层叠等级，也会创立一个新的层叠上下文(应该是为他的后代)

The stack level of the generated box in the current stacking context is 0. If the box has ‘position: fixed’ or if it is the root, it also establishes a new stacking context.

在当前层叠上下文中 盒子的层叠等级为0.如果盒子的定位为fixed或者他是根元素（应该是html） 那么他也会j建立一个新的层叠上下，文否则不建立层叠上下文
注意：z-index值为 0 与z-index值为auto的区别。区别是前者对内创建一个新的层叠上下文，后者不创建新的层叠上下文。除了这点区别，两者的层级是一样的。会根据文档树的顺序决定谁先 谁后

In this section, the expression “in front of” means closer to the user as the user faces the screen.

在这部分，“in front of‘表达式的意思是 当用户面对屏幕时，更接近于用户。

In CSS 2.2, each box has a position in three dimensions. In addition to their horizontal and vertical positions, boxes lie along a “z-axis” and are formatted one on top of
the other. Z-axis positions are particularly relevant when boxes overlap visually. This section discusses how boxes may be positioned along the z-axis.

在css2.2中 每个盒子都有一个在三维中的位置，除了水平和垂直位置外，盒子会沿着“z轴”排列，并且格式化为一个盒子在另一个盒子之上。当盒子在视觉上重叠时，z轴位置特别相关。本节讨论，盒子是如何被放置在z轴上的。

The order in which the rendering tree is painted onto the canvas is described in terms of stacking contexts. Stacking contexts can contain further stacking contexts. A stacking context is atomic from the point of view of its parent stacking context; boxes in other stacking contexts may not come between any of its boxes.

在画布上绘制渲染树的顺序是用叠加上下文来描述的。层叠上下文可以包含更多层叠上下文。从一个层叠上下文的父级层叠上下文的角度来看，该堆叠上下文是原子的；在其他层叠上下文中的盒子可能不会出现在它（应该是前面的那个一个层叠上下文）的任何一个盒子之间。

Each box belongs to one stacking context. Each positioned box in a given stacking context has an integer stack level, which is its position on the z-axis relative other stack levels within the same stacking context. Boxes with greater stack levels are always formatted in front of boxes with lower stack levels. Boxes may have negative stack levels. Boxes with the same stack level in a stacking context are stacked back-to-front according to document tree order.

每个盒子都属于一个层叠上下文，在给定层叠上下文中的每个定位盒子都有一个层叠等级，这个决定了它在z轴上的位置，相对于同一层叠上下文中的其他层叠等级。更大层叠等级中的盒子总是比更低层叠等级中的盒子被格式化的更接近用户。盒子可以有负的层叠等级。在同一个层叠上下文中拥有相同层叠等级的盒子会根据文档树的顺序前后堆叠

The root element forms the root stacking context. Other stacking contexts are generated by any positioned element (including relatively positioned elements) having a computed value of ‘z-index’ other than ‘auto’. Stacking contexts are not necessarily related to containing blocks. In future levels of CSS, other properties may introduce stacking contexts, for example ‘opacity’ [CSS3COLOR].

根元素会形成根层叠上下文。其他层叠上下文 通过一个 z-index的计算值不为auto的定位元素生成（包括相对定位）。堆叠上下文不一定与包含块有关。

Within each stacking context, the following layers are painted in back-to-front order:

在每一个层叠上下文中，以下的图层都是按先后顺序绘制的:

  1. the background and borders of the element forming the stacking context.

    构成层叠上下文的元素的背景和边界。

  2. the child stacking contexts with negative stack levels (most negative first).

    和负层叠等级的子层叠上下文(最大的负数优先)

  3. the in-flow, non-inline-level, non-positioned descendants.

    在文档流中的（in-flow），非行内级的（non-inline-level），非定位（non-positioned）的后代元素

  4. the non-positioned floats.

    非定位的浮动元素

  5. the in-flow, inline-level, non-positioned descendants, including inline tables and inline blocks.

    在文档流中的（in-flow），行内级的（inline-level），非定位（non-positioned）的后代元素，包括行内块级元素（inline blocks）和行内表格元素（inline tables）

  6. the child stacking contexts with stack level 0 and the positioned descendants with stack level 0.

    堆叠层级为 0 的子堆叠上下文（child stacking contexts）和堆叠层级为 0 的定位的后代元素

  7. the child stacking contexts with positive stack levels (least positive first).

    拥有正的层叠等级的子层叠上下文（最小的整数 优先）

Within each stacking context, positioned elements with stack level 0 (in layer 6), non-positioned floats (layer 4), inline blocks (layer 5), and inline tables (layer 5), are painted as if those elements themselves generated new stacking contexts, except that their positioned descendants and any would-be child stacking contexts take part in the current stacking context.

就好像这些元素本身产生了新的堆叠上下文，除了它们的定位后代 和 任何可能的子层叠上下文 会参与当前的堆叠上下文。（新的层叠上细纹只是好像产生了，实际上并没有产生，所以他的定位后代 和 子层叠上下文都会参与到当前的层叠上下文中）

This painting order is applied recursively to each stacking context. This description of stacking context painting order constitutes an overview of the detailed normative definition in Appendix E.

此绘制顺序将递归应用于每个层叠上下文。 层叠上下文绘制顺序的这种描述构成了附录E中详细的规范性定义的概述。

The default behavior of the background is to allow boxes behind it to be visible. In the example, each box transparently overlays the boxes below it. This behavior can be overridden by using one of the existing background properties.

背景的默认行为是允许该盒背后的盒可见。 在该示例中，每个框都透明地覆盖它下面的框。 这种行为可以通过使用现有的背景属性来改写覆盖。
