发表于 2018-04-25 | 更新于 2018-05-04

常用字符操作:

  • strObj.charAt(index) 返回 index对应位置的字符

  • strObj.indexOf(字符|正则) 返回下标 没有则返回-1

  • strObj.lastIndexOf(字符) 返回下标 没有则返回-1

  • strObj.toLowerCase() 返回 字符串小写

  • strObj.toUpperCase() 返回 字符串大写

  • strObj.substring(start,end) 返回指定范围内的字符 区间[start, end)不包含end

  • strObj.replace(要替换，替换成) 返回替换后的完整字符串

  • strObj.replace(正则,function(s){}) s = 每次匹配到的字符,在函数里面处理，之后return出来的数据被replace方法接收,拿去做替换

  • strObj.split(规律|正则) 按照规律返回数组

  • strObj.charCodeAt(index)
    返回指定位置的字符的Unicode编码,idnex指定的位置;
    ‘abc’.charCodeAt(0) === 97;
    index超出范围返回NaN ‘abc’.charCodeAt(4) // NaN

  • String.fromCharCode(code)
    将 Unicode 编码转为一个字符:String.fromCharCode(65) === ‘A’

  • strObj.search(字符|正则) 返回 下标 匹配不到返回-1

  • strObj.match(字符|正则) 返回 数组 匹配不到返回null

  • reObj.test(字符) 返回 true/false
    test: 左到右,整段字符只要有一部分符合,返true,并停下来,如果有g，会有一个lastIndex属性记住上一次匹配到的位置

转义符： \

作用：

不管是正则中的\ 还是字符串中的\ 后面的字符能转义的就转义,不能转义的 \就会消失;

比如 字符串’\na’此时n被\转义了,表示换行,’\y’由于y无法被转义,所以\等于消失了 ‘\y’===’y’;

正则中也是如此 /\ba/ b被转移成单词边界,字符串中\b被转义成退格 例如：

//正则中的d被转义成匹配数字 字符串中的d没有被转义还是字符d

/\d/.test('d')  //false
/\d/.test('\d') //false
/d/.test('\d')  //true
/\d/.test('123') //true

new RegExp(‘规则字符串’,’选项’);这种方式创建正则,字符串里的元字符 需要双重转义;
例如’\w’两次转义，字符串的时候转义一次，正则的时候还会转义一次

字面量模式                等价的字符串

/\w/                    "\\w"

/\[java\]script/        "\\[java\\]script"

/\.com/                "\\.com"

/China\/Eng/            "China\\/Eng"

/\d\.\d{1,2}/            "\\d\\.\\d{1,2}"

/\w\\hello/            "\\w\\\\hello"

原因是因为 字符串中 \ 会尝试转义后面的字符 所以要用 前面的\ 把 后面的\ 转义

字符转义： 为了描述

'a\b'    b被转义  --> 退格
'a\nb'    n被转义了    -->    换行

正则也有转义： 为了简化

    \d    数字    [0-9]

match和exec方法

match：是字符串的方法
exec: 是正则的方法 如果找到了匹配的文本，则返回一个结果数组。否则，返回 null。

正则中没有g的情况：

reg.exec(str) 和 str.match(reg) 2者的返回值是相同的
返回值都是： [正则完整匹配到的内容,(匹配到的分组内容,分组内容,分组内容…,),index,输入的字符串]

正则中如果有g的情况 ：

exec方法:

正则对象会有一个lastIndex属性,初始值为0；
正则会从lastIndex指定的位置开始匹配字符串,如果匹配到内容,会把lastIndex设置为匹配到的字符串的最后一个字符的下一个位置；
下次再调用reg.exec(str)的时候,就从lastIndex开始匹配;
当 exec() 再也找不到匹配的文本时，它将返回 null，并把 lastIndex 属性重置为 0。再调用的时候又从0开始匹配

match方法:

进行全局匹配 并把匹配到的所有子串作为数组内容返回 如果没有匹配到 返回null;

search方法

返回第一个满足条件的匹配结果在原字符串中的位置，如果没有匹配，则返回-1
"_x_x".search(/x/)
注：该方法会忽略g修饰符
var r= /x/g 
r.lastIndex =2; // seacrh方法设置lastindex属性是无效的
console.log('_x_x'.search(r)) //1

贪婪模式:

1、在匹配成功的前提下，尽可能多的去匹配

2、量词符： * + ? {n, m} {n,}

3、默认情况下都是最大可能匹配，即匹配直到下一个字符不满足匹配规则为止。这被称为贪婪模式.

例如:

惰性模式

1、在匹配成功的前提下，尽可能少的去匹配

2、量词符： *? +? ?? {n, m}? {n,}?

*? 重复任意次，但尽可能少重复
+? 重复1次或更多次，但尽可能少重复
?? 重复0次或1次，但尽可能少重复
{n,m}? 重复n到m次，但尽可能少重复
{n,}? 重复n次以上，但尽可能少重复

后向引用

使用小括号指定一个子表达式(分组)后，匹配到这个子表达式的文本后，可以在表达式或其它程序中作进一步的处理。
默认情况下，每个分组会自动拥有一个组号，规则是：从左向右，以分组的左括号为标志，第一个出现的分组的组号
为1，第二个为2，以此类推。

后向引用用于重复搜索前面某个分组匹配的文本。如过要重复搜索第一个分组匹配到的内容，可使用 \1。

例如：

/\b(\w+)\b\s+\1\b/ : 可以用来匹配重复的单词，像go go, 或者kitty kitty。

拆分理解：

\b(\w+)\b : 该表达式会匹配一个单词 然后这个单词(\w+)会被分配到一个编号为1的分组中

\s+: 会匹配1个或多个空格

\1\b : \1代表分组1中匹配到的内容（也就是前面匹配的那个单词）\b代表\1就是一个单词，后面不再跟其他字母

str = 'hello hello'

reg = /\b(\w+)\b\s+\1\b/g

str.match(reg) // ["hello hello"]

你也可以自己指定子表达式的组名。要指定一个子表达式的组名，请使用这样的语法：(?<Word>\w+)(或者把尖括号换成''也行：(?'Word'\w+)), 这样就把\w+的组名指定为Word了。要后向引用这个分组捕获的内容，你可以使用\k<Word>,所以上一个例子也可以写成这样：/\b(?<Word>\w+)\b\s+\k<Word>\b/。

常用分组语法：

捕获：

1、(exp)    匹配exp,并捕获文本到自动命名的组里

2、(?<name>exp)    匹配exp,并捕获文本到名称为name的组里，也可以写成(?'name'exp)

3、(?:exp)    匹配exp,不捕获匹配到的文本到分组里，也不会分配组号

第三个(?:exp)匹配到的内容并不会分组，也不会分配组号，当你需要匹配一段重复的内容，但是又不想给他分组的时候可以使用。

例如：
    var str = 'abcabcabc'; 
    var reg = /(?:abc)+/g; // 使用(?:exp)
    reg.exec(str)
    // ["abcabcabc", index: 0, input: "abcabcabc", groups: undefined]

    var reg = /(abc)+/g; // 不使用(?:exp)
    reg.exec(str);
    // ["abcabcabc", "abc", index: 0, input: "abcabcabc", groups: undefined]

零宽断言：

1、(?=exp)    匹配exp前面的位置

2、(?<=exp)    匹配exp后面的位置

3、(?!exp)    匹配后面跟的不是exp的位置

4、(?<!exp)    匹配前面不是exp的位置

零宽断言

上面四个语法用于查找在某些内容(但并不包括这些内容)之前或之后的东西，也就是说它们像\b,^,$那样用于指定一个位置，这个位置应该满足一定的条件(即断言)，因此它们也被称为零宽断言。

零宽断言只匹配一个位置，不匹配任何字符串。

(?=exp)：
也叫零宽度正预测先行断言，它断言自身出现的位置的后面能匹配到表达式exp，但是匹配结果中不包含exp。

比如\b\w+(?=ing\b)，匹配以ing结尾的单词的前面部分(除了ing以外的部分)，如查找I’m singing while you’re dancing.时，它会匹配sing和danc。

(?<=exp)：也叫零宽度正回顾后发断言，它断言自身出现的位置的前面能匹配表达式exp, 匹配结果不包含exp。

比如(?<=\bre)\w+\b会匹配以re开头的单词的后半部分(除了re以外的部分)，例如在查找reading a book时，它匹配ading。
